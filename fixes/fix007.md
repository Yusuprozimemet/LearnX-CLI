# fix007 — Empty SRT caused ffmpeg to abort subtitle embedding

**Date:** 2026-05-09

## Symptom

```
Invalid data found when processing input
```

ffmpeg crashed when trying to embed the subtitle track because the SRT file
existed but contained only a newline (size < 20 bytes).

## Root cause

`tutorial.units.json` had no `lines` field, so `build_srt([])` wrote an empty
file. The `_embed_subtitles()` call then passed the empty file to ffmpeg, which
rejected it as an invalid subtitle stream.

Two separate sub-bugs:
1. `_load_all_lines()` returned `[]` when the units JSON had no `lines` field
   and no fallback parser existed.
2. `assemble_session()` always called `_embed_subtitles()` regardless of SRT
   content.

## Files changed

- `tutor/visual/__init__.py` — `_load_all_lines()`
- `tutor/visual/video_assembler.py` — `assemble_session()`

## What changed

1. `_load_all_lines()` gained a fallback: when the JSON has no `lines` field it
   parses `tutorial.script.txt` with the regex `^(ALEX|MAYA|SAM):\s*(.+)$` and
   distributes lines evenly across units.

2. `assemble_session()` now guards the embed call:

```python
if srt_path.exists() and srt_path.stat().st_size > 20:
    _embed_subtitles(nosub, srt_path, final)
else:
    nosub.rename(final)
```
