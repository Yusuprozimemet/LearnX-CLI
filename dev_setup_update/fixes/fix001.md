# fix001 — Three Dockerfile gotchas when building learnx-dev on Python 3.12

**Date:** 2026-05-14

## Symptom

`docker build -t learnx-dev .` failed at the `pip install` step with:

```
ERROR: No matching distribution found for audioop-lts>=0.2.1
```

After fixing that, `pytest` still produced 7 failures in `test_slide_renderer.py`:

```
playwright._impl._errors.Error: BrowserType.launch: Executable doesn't exist at
/home/dev/.cache/ms-playwright/chromium_headless_shell-1217/...
```

And `ruff` was missing entirely:

```
/usr/local/bin/python: No module named ruff
```

## Root causes

### 1. `audioop-lts` has no distribution for Python 3.12

`tutor/requirements.txt` contains:

```
audioop-lts>=0.2.1  # pydub compatibility for Python 3.13+
```

The comment says it all — this package is a backport of the `audioop` stdlib module
that was removed in Python 3.13. PyPI has no wheel for Python 3.12 because `audioop`
is still in the standard library there. The Dockerfile targets Python 3.12-slim, so
pip cannot satisfy the requirement and fails immediately.

### 2. Playwright Python package ≠ Playwright browser binary

Installing `playwright>=1.44` via pip installs the Python bindings only. The Chromium
browser executable must be downloaded separately with `playwright install chromium`.
That command also needs OS-level system libraries (`libatk`, `libgdk`, etc.) which
require a separate `playwright install-deps chromium` step run as root before the
user-level binary install.

Without this, every test that calls `sync_playwright().chromium.launch()` raises the
"Executable doesn't exist" error at runtime even though `import playwright` succeeds.

### 3. `ruff` is not in `tutor/requirements-dev.txt`

`tutor/requirements-dev.txt` only lists `pytest` and `pytest-asyncio`. `ruff` is
defined in `pyproject.toml` under `[project.optional-dependencies] dev` but was never
added to the `requirements-dev.txt` file. Since the Dockerfile installs from the
requirements files (not from `pyproject.toml`), `ruff` is absent from the image.

- `Dockerfile` (new)

## What changed

```dockerfile
# Filter out audioop-lts before installing — not available for Python 3.12
RUN grep -v audioop-lts /tmp/requirements.txt > /tmp/requirements-filtered.txt && \
    pip install --no-cache-dir \
    -r /tmp/requirements-filtered.txt \
    -r /tmp/requirements-dev.txt \
    "playwright>=1.44" \
    "jinja2>=3.1" \
    "Pillow>=10.0" \
    "ruff>=0.4"

# Install Playwright system dependencies (requires root)
RUN python -m playwright install-deps chromium

USER dev

# Install Playwright browser binary for the dev user
RUN python -m playwright install chromium
```

The three rules:
1. Filter `audioop-lts` from requirements when building a Python 3.12 image.
2. Run `playwright install-deps chromium` as root, then `playwright install chromium`
   as the non-root user, both during image build — not at container start time.
3. Add `ruff>=0.4` explicitly to the pip install list; do not assume it comes from
   `requirements-dev.txt`.
