# fix004 — `--verbose` / `--debug` not recognised by `/generate`

**Date:** 2026-05-08

## Symptom

```
LearnX > /generate notes.md --verbose
generate: error: unrecognized arguments: --verbose
Error: could not parse arguments.
```

## Root cause

`_make_generate_parser()` was extracted from the top-level CLI parser but `--verbose`
and `--debug` were not included. Also, `logging.basicConfig()` is a one-shot call —
once the shell starts it has already been called, so calling `_setup_logging(args)` per
command has no effect.

## Files changed

- `tutor/tutor.py` — `_make_generate_parser()`
- `tutor/cli/commands.py` — `cmd_generate`, new `_apply_log_level()`

## What changed

1. Added `--verbose` and `--debug` to `_make_generate_parser()`.
2. Added `_apply_log_level(args)` in `commands.py` that calls
   `logging.getLogger().setLevel()` directly on the live root logger — works at any
   point in the process, unaffected by `basicConfig` having already been called.
3. `cmd_generate` in the shell calls `_apply_log_level(args)` after parsing.
