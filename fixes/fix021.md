# fix021 — CI unit-tests fails: jinja2 missing and slow tests crash without browser

**Date:** 2026-05-14

## Symptom

After fix020 unblocked the `pip install` step, CI unit-tests failed at collection:

```
ImportError while importing test module 'tutor/tests/visual/test_slide_renderer.py'
ModuleNotFoundError: No module named 'jinja2'
```

No tests ran at all — pytest aborts on a collection-time ImportError.

## Root causes

### 1. `jinja2` and `playwright` missing from `tutor/requirements.txt`

`tutor/visual/slide_renderer.py` imports both at module level:

```python
from jinja2 import Environment, FileSystemLoader
from playwright.sync_api import sync_playwright
```

`test_slide_renderer.py` also imports `from jinja2 import TemplateNotFound` at the top.
Because these are module-level imports, pytest fails during collection — before any test
function runs — if the packages are absent.

Both packages are declared in `pyproject.toml` under `[project.dependencies]`, but CI
installs from `tutor/requirements.txt` (not from `pyproject.toml`). They were never
added to `requirements.txt`.

**Fix:** added `jinja2>=3.1` and `playwright>=1.44` to `tutor/requirements.txt`.

### 2. `@pytest.mark.slow` tests need a Playwright browser binary

Several tests in `test_slide_renderer.py` are marked `@pytest.mark.slow` and call
`render_all_slides()`, which launches a real Playwright Chromium browser to take
screenshots. The `unit-tests` CI job installs the `playwright` Python package but does
not run `playwright install chromium`, so the browser binary is absent.

If these tests were allowed to run, each would fail with:
```
playwright._impl._errors.Error: BrowserType.launch: Executable doesn't exist
```

The `e2e-tests` job does run `playwright install chromium --with-deps`, but it only
executes `tutor/tests/e2e/` — the visual slow tests never run there.

**Fix:** added `-m "not slow"` to the `unit-tests` pytest command. The non-slow tests
in `test_slide_renderer.py` use mocks and do not need a browser — they run fine without
Playwright installed as a binary.

## Files changed

- `tutor/requirements.txt` — added `jinja2>=3.1` and `playwright>=1.44`
- `.github/workflows/ci.yml` — added `-m "not slow"` to unit-tests command

## Rules for future work

1. Any runtime import in `tutor/` must be reflected in `tutor/requirements.txt`.
   `pyproject.toml` is not used by CI.
2. Tests that require a Playwright browser binary must be marked `@pytest.mark.slow`.
   The `unit-tests` CI job skips them (`-m "not slow"`); they run only when a browser
   is available (local or e2e job).
3. After adding a new dependency anywhere in `tutor/`, update both `pyproject.toml`
   and `tutor/requirements.txt`.
