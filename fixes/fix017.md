# fix017 — `_load_unit_lines` skips all units when `tutorial.units.json` has no `lines` field

## Symptom

Running `/video` after `/generate` produced a 0 MB, 0:10 MP4 (title card + outro only).
The log showed:

```
WARNING  tutor.generation.segment_planner: Unit 1 has no dialogue lines — skipping
WARNING  tutor.generation.segment_planner: Unit 2 has no dialogue lines — skipping
...
```

`tutorial.segments.json` was written with `"units": {}`.

## Root Cause

`segment_planner._load_unit_lines()` read `tutorial.units.json` and built
`DialogueLine` objects from each unit's `"lines"` key. But `tutorial.units.json`
is written by the generate pipeline as:

```python
units_path.write_text(json.dumps([asdict(u) for u in units], ...), ...)
```

where `units` is a list of `TeachingUnit` dataclass objects. `TeachingUnit` has
**no `lines` field** — dialogue lives in `tutorial.script.txt`. So every unit's
`"lines"` key was either absent or `[]`, and `_load_unit_lines` returned all units
with empty line lists, which `plan_segments` skipped.

Unit-test fixtures in `tests/` explicitly included `"lines"` in their JSON, masking
the bug completely.

## Fix

Added a two-stage fallback in `_load_unit_lines()` (committed to main via Day 16):

**Stage 1** — after reading `units.json`, if all units still have zero lines,
fall back to parsing `tutorial.script.txt` for `(speaker, text)` pairs.

**Stage 2** — distribute those pairs across units using `tutorial.timing.json`
line counts (most accurate), or proportional split (fallback):

```python
if any(len(lines) > 0 for _, lines in result.values()):
    return result   # units.json had real lines — done

script_path = units_json_path.parent / "tutorial.script.txt"
# ... parse ALEX/MAYA/SAM lines ...

timing_path = units_json_path.parent / "tutorial.timing.json"
# ... read lines_per_unit from timing.json ...

if lines_per_unit and all(u in lines_per_unit for u in result):
    # Timing-based slice: skip n_intro intro lines, then slice per-unit counts
    n_teaching = sum(lines_per_unit.values())
    n_non = max(0, len(all_pairs) - n_teaching)
    n_intro = (n_non + 1) // 2
    cursor = n_intro
    for unit_num in sorted(result.keys()):
        count = lines_per_unit[unit_num]
        result[unit_num] = (concept, [...slice of all_pairs...])
else:
    # Proportional fallback
    per_unit = max(1, len(all_pairs) // max(n_units, 1))
    ...
```

## Result

For `week2/2.md` (4 teaching units, 43 total script lines, 41 teaching lines):
- Before fix: 0 MB, 0:10 MP4 (no unit slides rendered)
- After fix: 7 MB, 8:12 MP4 (all 4 unit slide stacks rendered)

## Files Changed

- `tutor/generation/segment_planner.py` — `_load_unit_lines()` extended with fallback
