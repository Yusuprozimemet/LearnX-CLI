# fix024 — Container prints "Claude configuration file not found" on every start

**Date:** 2026-05-15

## Symptom

Every container session printed this warning three times before the Claude Code
prompt appeared:

```
Claude configuration file not found at: /home/dev/.claude.json
A backup file exists at: /home/dev/.claude/backups/.claude.json.backup.<timestamp>
You can manually restore it by running: cp "..." "/home/dev/.claude.json"
```

Claude Code still started, but the repeated warnings were confusing and the
backup timestamp changed on every run.

## Root cause

Claude Code looks for its main config file at `~/.claude.json` — that is, in the
user's home directory, **not** inside the `~/.claude/` subdirectory.

The container mounts only the `.claude` directory:

```python
cmd += ["-v", f"{_to_posix(claude_dir)}:/home/dev/.claude:ro"]
```

The `.claude.json` file lives one level up at `C:\Users\<user>\.claude.json` on
the host. That file was never mounted into the container, so Claude Code could not
find it, fell back to the backup inside `~/.claude/backups/`, and printed the
warning.

## Fix

Mount `.claude.json` separately, read-only:

```python
claude_json = home_dir / ".claude.json"
...
if claude_json.exists():
    cmd += ["-v", f"{_to_posix(claude_json)}:/home/dev/.claude.json:ro"]
```

This is a conditional mount — if the file does not exist on the host the
container starts without it, which is the same behaviour as before.

## Files changed

- `scripts/learnx_dk.py` — `build_docker_command()`: added conditional mount
  for `~/.claude.json`

## Rules for future work

1. Claude Code keeps two separate config locations:
   - `~/.claude/`       — directory: settings, agents, session state
   - `~/.claude.json`   — file: main config (auth tokens, preferences)
   Both must be mounted for a complete container environment.
2. Both mounts should remain `:ro`. The container should never write to host
   Claude config files.
