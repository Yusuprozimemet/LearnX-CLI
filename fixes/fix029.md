# fix029 — OpenRouter free model churn: models disappear or fail at JSON

**Date:** 2026-05-15

## Symptom

Several OpenRouter free models that appeared in the model list failed in different ways:

| Model | Failure |
|---|---|
| `google/gemma-3-27b-it:free` | 404 — removed from platform |
| `deepseek/deepseek-v4-flash:free` | Returns 200 OK but invalid JSON for structured calls |
| `arcee-ai/trinity-large-thinking:free` | Returns 200 OK but empty `message.content` |
| `nvidia/nemotron-3-nano-omni:free` | 400 — bad request (wrong endpoint or model config) |

The free tier on OpenRouter has high model churn — models are added and removed
frequently, and not all models support the same output constraints.

## Root cause

Free-tier models on OpenRouter are community-hosted or provider-sponsored endpoints.
They can be withdrawn without notice. Additionally:

- **Flash/distilled models** (deepseek-v4-flash) are optimised for speed, not
  instruction following — they frequently ignore JSON-only instructions and add prose.
- **Thinking models** (trinity-large-thinking) output their reasoning into a non-standard
  field (`reasoning_content`) and return empty `content` — the OpenAI SDK returns
  `None` for `message.content`, causing an assertion failure.
- **VL / multimodal models** may require different request formats for text-only use.

## Fix

Added `scripts/test_openrouter_models.py` — a benchmark script that tests every model
in `tutor/models.md` for:
1. JSON output validity (curriculum gate)
2. Dialogue format compliance (SPEAKER: text lines)

Run before switching models:

```powershell
py scripts/test_openrouter_models.py --quick    # JSON check only, ~1 min
py scripts/test_openrouter_models.py            # full check including dialogue
```

Working model as of 2026-05-15: **`poolside/laguna-xs.2:free`** — 1.9s JSON response,
passes both checks, handles all call types.

## Rules for future work

1. Run the benchmark script whenever you want to switch OpenRouter models.
2. A 404 means the model ID is wrong or the model was removed — check
   OpenRouter's model list and update `tutor/models.md`.
3. A 400 does not always mean wrong ID — some models have restricted endpoints.
4. Flash/distilled models fail the JSON gate almost universally — avoid them for
   curriculum and visual planner calls even if they pass dialogue.
5. Thinking models need special handling: strip `<think>...</think>` from content,
   and check that `message.content` is not None before using it.
6. Update `tutor/models.md` periodically with the current OpenRouter free model list.
