# fix008 — Unicode characters crashed Windows CP1252 console

**Date:** 2026-05-09

## Symptom

```
UnicodeEncodeError: 'charmap' codec can't encode character '✓' in position 4
```

The pipeline printed `✓` (U+2713 CHECK MARK) on completion; Windows console
defaults to CP1252 which cannot encode it.

## Root cause

Several `print()` calls in `video_assembler.py` used Unicode symbols that are
not representable in Windows-1252. Similarly, `slide_compositor.py` used curly
quotes `"` (U+201C) and `■` (U+25A0) in draw calls; Segoe UI Bold does not
reliably include U+201C in all Windows installations, so it rendered as boxes.

## Files changed

- `tutor/visual/video_assembler.py`
- `tutor/visual/slide_compositor.py`
- `tutor/cli/shell.py` — `_setup_utf8()` added as a belt-and-suspenders fix

## What changed

1. `video_assembler.py`: `print(f"  OK  {final} ...")` — `✓` replaced with
   plain text `OK`.
2. `slide_compositor.py`: curly open-quote `"` replaced with `//`; `■`
   replaced with `-`.
3. `shell.py`: `_setup_utf8()` wraps `sys.stdout` / `sys.stderr` with
   `encoding="utf-8", errors="replace"` at startup so any remaining Unicode in
   CLI output degrades gracefully instead of crashing.
