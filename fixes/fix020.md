# fix020 — CI unit-tests and e2e-tests jobs fail on audioop-lts install (Python 3.12)

**Date:** 2026-05-14

## Symptom

Both `unit-tests` and `e2e-tests` CI jobs failed immediately at the "Install dependencies"
step with:

```
ERROR: Could not find a version that satisfies the requirement audioop-lts>=0.2.1
ERROR: No matching distribution found for audioop-lts>=0.2.1
Error: Process completed with exit code 1.
```

No tests ran. The lint job passed; only the two jobs that install `tutor/requirements.txt`
were affected.

## Root cause

`audioop-lts` is a backport of the `audioop` stdlib module that was removed in Python 3.13.
PyPI publishes wheels only for Python 3.13+ because Python 3.12 still ships `audioop` in
its standard library — there is nothing to backport.

`tutor/requirements.txt` includes:

```
audioop-lts>=0.2.1  # pydub compatibility for Python 3.13+
```

The comment is correct — this line is only needed on Python 3.13+. But the CI jobs run
on Python 3.12 and install `requirements.txt` verbatim, so pip fails immediately.

This is the same root cause documented in **fix018** (Dockerfile build failure).
fix018 solved it for the Docker image; CI was not covered because it was added after
the Dockerfile fix and the same filter was not carried over.

## Why it wasn't caught locally

The local `.venv` was created before `audioop-lts` was added to `requirements.txt`, so
`pip install -r requirements.txt` was never re-run against the current file in the
existing venv. A fresh `pip install -r tutor/requirements.txt` on Python 3.12 Windows
would show the same error.

**Rule: always run `pip install -r tutor/requirements.txt` from scratch (or in a clean
venv) before pushing, especially after any requirements change.** CI is a clean
environment every time — the local venv hides stale installs.

## Fix

Filter `audioop-lts` from `requirements.txt` before passing it to pip, in both CI jobs
that install Python dependencies — identical to the Dockerfile pattern from fix018:

```yaml
- name: Install dependencies
  run: |
    grep -v 'audioop-lts' tutor/requirements.txt > /tmp/requirements-filtered.txt
    pip install -r /tmp/requirements-filtered.txt
    pip install -r tutor/requirements-dev.txt
```

Applied to `unit-tests` ("Install dependencies") and `e2e-tests` ("Install Python
dependencies") in `.github/workflows/ci.yml`.

## Files changed

- `.github/workflows/ci.yml`

## Rule for future CI jobs

Any job that installs `tutor/requirements.txt` on Python 3.12 must filter `audioop-lts`
first. Copy the `grep -v` pattern above verbatim. Do not install the file directly with
`pip install -r tutor/requirements.txt`.
