# fix023 — yolo E2E tests fail on host: ffmpeg and Playwright not installed on Windows

**Date:** 2026-05-15

## Symptom

After exiting the container session, the yolo Step 2 (E2E tests) failed immediately
for every test:

```
tutor.exceptions.ConfigError: ffmpeg not found in PATH.
  Install with: winget install ffmpeg
  Or add its bin\ folder to your system PATH and restart the terminal.
```

22 errors, 0 tests ran.

## Root cause

`run_yolo()` originally ran the E2E tests on the Windows host:

```python
e2e_cmd = ["python", "-m", "pytest", "tutor/tests/e2e/", "-v"]
```

The E2E test suite calls the real pipeline via `conftest.py`, which calls
`preflight()` → `_check_ffmpeg()`. ffmpeg was not installed on the Windows
host machine.

The same issue applies to Playwright: E2E tests that render slides require
the Chromium browser binary, which is also not present on the host. On Windows,
the `greenlet` DLL that Playwright depends on also fails to load.

The `learnx-dev` Docker container already has ffmpeg, Playwright, and the
Chromium binary installed. Running E2E on the host was the wrong location.

## Fix

Run E2E tests inside the container (no `-it` flag, no claude entrypoint):

```python
def _build_e2e_command(project_dir: pathlib.Path) -> list[str]:
    return [
        "docker", "run", "--rm",
        "-v", f"{_to_posix(project_dir)}:{WORKSPACE}",
        "-w", WORKSPACE,
        IMAGE,
        "python", "-m", "pytest", "tutor/tests/e2e/", "-v",
    ]
```

The container has all required binaries. The workspace volume mount gives it
access to `tutor/.env` (GROQ API key) and the test files.

## Files changed

- `scripts/learnx_dk.py` — `run_yolo()`: replaced host `e2e_cmd` with
  `_build_e2e_command()` that runs pytest inside the container

## Rules for future work

1. E2E tests must always run in an environment that has ffmpeg, Playwright,
   and the Chromium binary. That environment is the `learnx-dev` container,
   not the Windows host.
2. Do not add E2E tests that depend on tools not in `learnx-dev`. If a new
   tool is needed, add it to the Dockerfile first.
3. The yolo review pipeline (Step 3) runs `run_review.py` on the host, which
   internally spawns its own container. That step is correct as-is.
