# fix012 — Hook slide held for 82 s because ALEX monologues before first MAYA line

**Date:** 2026-05-09

## Symptom

The hook slide stayed on screen for the entire ALEX pre-concept monologue
(82 s for unit 1) before transitioning to the concept slide. This left the
simple hook question frozen on screen for far too long.

## Root cause

`_build_beat_map()` defines the hook slide duration as:

```python
hook_dur = concept_t - hook_t
```

`concept_t` is the timestamp of the **first MAYA line** in the unit. ALEX
introduces the hook question and then continues talking for several exchanges
before MAYA joins. Any ALEX-only preamble is counted as hook-slide time,
which can easily exceed 60–80 s in a 4-unit session.

## Files changed

- `tutor/visual/beat_timer.py`

## What changed

Added `MAX_HOOK_DURATION = 30.0` seconds. The hook slide is capped at this
value; any surplus time is donated to the concept slide:

```python
raw_hook    = concept_t - hook_t
hook_dur    = _clamp(min(raw_hook, MAX_HOOK_DURATION))
concept_dur = _clamp(memory_t - concept_t + max(0.0, raw_hook - MAX_HOOK_DURATION))
```

Result for unit 1: hook 30 s (was 82 s), concept 164 s (was 112 s). The total
unit audio coverage is unchanged; the time is simply redistributed.
