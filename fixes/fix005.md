# fix005 — Slide fonts render at 8 px (bitmap fallback) on Windows

**Date:** 2026-05-09

## Symptom

All text on generated slides appeared microscopic (8 px Pillow bitmap font).
Unicode characters (·, ⬡, 📌, curly quotes) rendered as garbled squares or
question-mark boxes. The design looked completely broken.

## Root cause

`ImageFont.truetype("Segoe UI", size)` fails silently on Windows when the font
is specified by name — Pillow on Windows requires the **full file-system path**
to the TTF file. All three sizes/weights fell through every fallback and hit
Pillow's built-in bitmap default (8 px fixed).

## Files changed

- `tutor/visual/slide_theme.py`

## What changed

`_load_font(size, bold, mono)` now follows a four-level cascade:

1. **Bundled TTF** — `tutor/assets/fonts/Inter-{Regular,Bold}.ttf` /
   `JetBrainsMono-{Regular,Bold}.ttf` (future-proof; dir exists but files not
   yet committed).
2. **Windows full path** — primary map: `C:/Windows/Fonts/segoeui.ttf`,
   `segoeuib.ttf`, `consola.ttf`, `consolab.ttf`; fallback map:
   `verdana.ttf`, `verdanab.ttf`, `cour.ttf`, `courbd.ttf`.
3. **Font by name** — works on Linux/macOS; harmless attempt on Windows.
4. **Bitmap default** — last resort, never `raise`.

Added `BG_PANEL = "#1c2128"` constant for bullet-card backgrounds.
