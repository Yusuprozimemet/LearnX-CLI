# fix006 — ffmpeg concat demuxer resolved paths relative to script directory

**Date:** 2026-05-09

## Symptom

```
Impossible to open 'video\week2_3\video\week2_3\slides\01_hook.png'
```

The path was doubled: the slide path appeared twice in the error.

## Root cause

`_write_concat_script()` was writing relative slide paths (e.g.
`video/week2_3/slides/01_hook.png`) into the concat script. ffmpeg's concat
demuxer resolves relative paths **relative to the concat script's own
directory** (not the process CWD). The script lives inside
`video/week2_3/`, so ffmpeg prepended that prefix again, doubling the path.

## Files changed

- `tutor/visual/video_assembler.py` — `_write_concat_script()`
- `tutor/tests/visual/test_video_assembler.py` — `test_concat_script_written_correctly`

## What changed

Each entry now uses the absolute, forward-slash-normalised path:

```python
abs_path = str(path.resolve()).replace("\\", "/")
lines.append(f"file '{abs_path}'")
```

The test was updated to compare against the resolved absolute path with the
same normalisation.
