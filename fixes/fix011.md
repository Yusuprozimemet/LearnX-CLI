# fix011 — Title card showed first unit's concept instead of document title

**Date:** 2026-05-09

## Symptom

Title card displayed "Interface vs Abstract Class" (the first teaching unit's
concept) instead of the actual document title. Every session showed a unit
concept rather than what the source document is about.

## Root cause

`_doc_title_from_units()` in `tutor/visual/__init__.py` read the first element
of `tutorial.units.json` and returned `units[0]["concept"]`. No document-level
title was stored anywhere in the pipeline output.

## Files changed

- `tutor/cli/commands.py` — `cmd_generate`
- `tutor/visual/__init__.py` — `_doc_title_from_units()`

## What changed

1. `cmd_generate` now writes `audio/<session>/tutorial.meta.json` on successful
   generation:
   ```json
   {"source_file": "week2/3.md"}
   ```

2. `_doc_title_from_units()` was rewritten with a three-level priority chain:
   - **H1/H2 heading from source file** — reads the markdown file at
     `meta["source_file"]`, strips leading numbering (`1. `, `2) `) and emoji,
     skips boilerplate headings ("Learning Objectives", "Introduction", etc.).
   - **Path-formatted filename** — if the stem is a digit (e.g. `3.md`) uses
     parent + stem: `week2/3.md` → `Week 2 - Part 3`. Otherwise title-cases
     the stem.
   - **First unit concept** — original behaviour, kept as last resort.
