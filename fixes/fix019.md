# fix019 — Ruff lint and format violations in E2E test suite (day5)

**Date:** 2026-05-14

## Symptom

`py -m ruff check tutor/tests/e2e/` and `py -m ruff format --check tutor/tests/e2e/`
failed after the initial day5 implementation with several categories of violations:

- Duplicate code: `sorted(SLIDE_DIR.glob("*.html"))` repeated four times in `test_slide_render.py`
- `float(value)` / `int(value)` called directly on ffprobe fields that can return `"N/A"` strings,
  causing `ValueError` at runtime and confusing ruff's analysis
- `# noqa: ARG001` suppressor on `_mock_llm` — ruff flagged the suppression as unnecessary
  after the function signature was changed, and suppressing lint instead of fixing it is fragile
- `with patch(...), patch.dict(...)` — old-style parenthesis-free `with` blocked ruff format
- Multi-line list / dict literals without trailing commas (ruff format E501 / trailing-comma rules)
- Missing blank line after module-level docstrings (ruff format D205 / W292 equivalent)
- `subprocess.run(["ffprobe", "-v", ...])` single-line arg list exceeded line length limit

## Root causes

### 1. Repeated glob pattern in `test_slide_render.py`

Four test functions each contained:

```python
html_files = sorted(SLIDE_DIR.glob("*.html"))
first_slide = html_files[0].as_uri()
browser_page.goto(first_slide)
```

Ruff's duplicate-code and line-length rules flagged this. Additionally, if `html_files` is empty
the bare `html_files[0]` raises `IndexError` with no diagnostic message.

**Fix:** extracted `_first_slide()` helper that asserts non-empty and returns the path.

### 2. `float(value)` / `int(value)` on ffprobe output

`ffprobe` returns `"N/A"` for `duration` and `bit_rate` when a stream has no timing metadata
(common for image-only or synthetic test videos). Calling `float("N/A")` raises `ValueError`,
crashing the test rather than producing a useful assertion failure.

The original code used `dict.get("duration", 0)` as a fallback, but that only guards against
a missing key — not against the `"N/A"` string value that ffprobe actually emits.

**Fix:** added `_to_float(value, default=0.0)` and `_to_int(value, default=0)` helpers that
catch `(TypeError, ValueError)` and return the default. The test assertion then fires with a
meaningful message instead of an unhandled exception.

### 3. `# noqa: ARG001` suppressor on `_mock_llm`

The original mock only handled `summarize` and `curriculum` call types and silently returned
`DIALOGUE_RESPONSE` for everything else (including unexpected types). The `**kwargs` parameter
was unused, hence the `ARG001` suppressor.

After cleanup, `**kwargs` was removed and the function raises `ValueError` for unknown
`call_type` values. With the suppressor gone, ruff no longer flags it, and any new LLM call
type that reaches the mock now fails loudly instead of returning wrong data silently.

### 4. Parenthesis-free `with` statement

```python
# before — ruff format does not accept this style in Python 3.10+ projects
with patch("tutor.infra.llm.chat", side_effect=_mock_llm), \
     patch.dict(os.environ, {"GROQ_API_KEY": "test-key-not-used"}):
```

Ruff format requires the parenthesised form for multi-context `with` statements:

```python
with (
    patch("tutor.infra.llm.chat", side_effect=_mock_llm),
    patch.dict(os.environ, {"GROQ_API_KEY": "test-key-not-used"}),
):
```

### 5. Multi-line literals missing trailing commas and blank lines after docstrings

Ruff format enforces trailing commas in multi-line list/dict literals and a blank line
after the module docstring. These were batch-fixed across all five e2e test files and
`conftest.py`.

### 6. `subprocess.run` single-line arg list

The `ffprobe_streams` helper had all arguments on one line, exceeding the 88-character
limit. Ruff format split it into one-argument-per-line style and also flagged the missing
`timeout` parameter (unbounded subprocess calls can hang CI).

**Fix:** reformatted to multi-line and added `timeout=30`. Added `result.returncode != 0`
check with `pytest.fail()` so a broken ffprobe install surfaces as a clear failure message
rather than a `JSONDecodeError` from empty stdout.

## Rules for future E2E tests

1. Never use `float(x)` or `int(x)` on ffprobe / ffmpeg output — always use a safe
   converter that handles `"N/A"`.
2. Never repeat a glob pattern more than once in the same file — extract a helper.
3. Never suppress a ruff rule with `# noqa` when fixing the underlying issue is possible.
4. Always use parenthesised `with (…):` form for multi-context managers.
5. Run `ruff format --check` before committing, not just `ruff check`.
