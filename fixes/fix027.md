# fix027 — Generation pipeline appeared frozen: no progress output

**Date:** 2026-05-15

## Symptom

Running `/generate` showed nothing after the command was entered. The shell appeared
frozen for several minutes with no feedback. It was impossible to tell whether the
pipeline was working, stuck, or had silently failed.

## Root cause

Every LLM step (summarise, curriculum, dialogue per unit) logged progress at `INFO`
level only. Without `--verbose`, the default log level is `WARNING`, so nothing was
printed. The first `print()` call in the pipeline only fired after ALL LLM work was
done — after summarise + curriculum + all dialogue units completed.

## Fix

Added explicit `print()` statements in `tutor/tutor.py` (`cmd_generate`) before each
major step, and per-unit progress during dialogue generation:

```python
print(f"Summarising {len(chunks)} chunk(s)...")
print("Planning curriculum...")
print(f"Generating dialogue for {len(units)} unit(s)...")
for i, u in enumerate(units, 1):
    print(f"  [{i}/{len(units)}] {u.concept}")
    all_lines.append(dialogue.generate(...))
```

These print regardless of `--verbose`. The user now sees exactly which step is running
and which unit is being generated, so a slow model is distinguishable from a hang.

## Rules for future work

1. If generation seems frozen, run with `--verbose` to see per-step INFO logs.
2. Any new pipeline step added to `cmd_generate` should have a `print()` before it —
   not just a `log.info()`.
3. Use `--script-only` to test the LLM steps without waiting for TTS rendering.
