Java traps to prioritise — pick those most relevant to the source content:
  is-a vs has-a (inheritance vs composition — the #1 OOP design mistake)
  super() must be the first statement in a subclass constructor
  @Override must always be used — silent typo override is a real production bug
  private fields are NOT inherited — access them only through getters
  overriding vs overloading — completely different mechanisms
  interface vs abstract class — when to choose which
  instanceof overuse — signals a missing polymorphism opportunity
  dynamic dispatch — the runtime decides the method, not the declared type
  checked vs unchecked exceptions — compiler enforces checked; unchecked are silent
  never swallow exceptions silently (empty catch block)
  equals() and hashCode() always go together — break one, break both
  == vs .equals() for Strings and objects
  pass-by-value confusion (references are passed by value)
  autoboxing surprises (Integer == Integer fails above 127)
  final ≠ immutable

Java concept map for reference:
Tier 0 — JVM mental model
  └─ bytecode, classloader, JIT, stack vs. heap

Tier 1 — Type system
  ├─ primitives vs. reference types
  ├─ pass-by-value (including references)
  └─ autoboxing / unboxing traps

Tier 2 — OOP mechanics
  ├─ constructors and this()
  ├─ inheritance: extends, super(), method overriding, @Override
  ├─ is-a (inheritance) vs has-a (composition)
  ├─ what is and is not inherited (private, constructors)
  ├─ method overloading vs overriding
  └─ final: variable / method / class differences

Tier 3 — Contracts and polymorphism
  ├─ interface as contract (implements, default, static methods)
  ├─ interface vs abstract class — decision criteria
  ├─ interface segregation (small, focused contracts)
  ├─ Comparable and Iterable from the standard library
  ├─ compile-time polymorphism (overloading)
  ├─ runtime polymorphism (dynamic dispatch, @Override)
  ├─ declaring variables by interface/superclass type
  └─ instanceof — correct use and when it signals a design problem

Tier 4 — Core contracts
  ├─ equals() and hashCode() contract
  ├─ == vs .equals() for Strings and objects
  └─ Comparable vs Comparator

Tier 5 — Collections
  ├─ List / Set / Map interfaces
  ├─ ArrayList vs LinkedList
  └─ HashMap internals (buckets, load factor)

Tier 6 — Error handling
  ├─ exception hierarchy: Throwable → Error / Exception → RuntimeException
  ├─ checked vs unchecked exceptions
  ├─ try-catch, finally, try-with-resources
  ├─ throw vs throws
  ├─ custom exception classes with domain context
  └─ best practices: specific catches, fail fast, never swallow silently

Tier 7 — Concurrency (advanced)
  ├─ thread safety and shared state
  ├─ synchronized and the monitor lock
  └─ volatile and visibility
