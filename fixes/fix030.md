# fix030 — poolside/laguna-xs.2:free occasionally returns empty content

**Date:** 2026-05-15

## Symptom

During dialogue generation with `poolside/laguna-xs.2:free`, some units triggered
this warning:

```
WARNING LLM call failed (LLM returned empty content), retrying in 2.0s...
```

The model returned HTTP 200 OK but `message.content` was `None` or an empty string.
The retry succeeded on the second attempt, so generation completed without error.

## Root cause

`laguna-xs.2` is a free-tier hosted model. Under load it occasionally returns an
empty response body — likely a backend timeout or resource limit on the provider's
side that still results in a 200 status code rather than a 503.

This is not a bug in LearnX. It is a reliability characteristic of free-tier inference.

## Fix

Two changes made to harden against this:

1. **Retry count raised from 2 to 3** in `tutor/llm_config.toml`:
   ```toml
   retry_count = 3   # was 2
   ```

2. **`<think>` stripping added** in `scripts/test_openrouter_models.py` so that
   thinking models (which hide content inside tags) are handled cleanly in the
   benchmark without masking empty-content failures.

The existing assertion in `llm.py` already catches this:
```python
assert content is not None, "LLM returned empty content"
```
This triggers the retry loop correctly.

## Rules for future work

1. Keep `retry_count = 3` when using free-tier OpenRouter models.
2. If empty-content failures become frequent (more than 1 per session), the model
   is under heavy load — switch to another passing model from the benchmark results.
3. The retry loop only helps for transient failures. If a model consistently returns
   empty content (every attempt), it is a structural issue with that model — see fix029.
4. Do not raise `retry_count` above 4 — it increases total wait time
   (`retry_count × retry_delay_s`) and a model that fails 4 times in a row is
   unlikely to recover on its own.
