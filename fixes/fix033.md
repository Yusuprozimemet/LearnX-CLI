# fix033 — --explore silently ignored --review and --version

**Date:** 2026-05-16

## Symptom

Passing `--explore --review` or `--explore --version v5` produced no error.
`main()` branched on `if explore:` first and returned early, silently discarding
`--review` or `--version`. The user got an explore session when they may have
expected an error or a different behaviour.

## Root cause

`_parse()` had no mutual-exclusion check for `--explore` vs `--review` / `--version`.
The early-return dispatch in `main()` happened to suppress the flags, but that is
invisible to the caller and misleading in scripts.

## Fix

Added two checks at the end of `_parse()`, after existing mutual-exclusion guards:

```python
if explore and review:
    print("error: --explore and --review are mutually exclusive")
    sys.exit(1)

if explore and version is not None:
    print("error: --explore and --version are mutually exclusive")
    sys.exit(1)
```

Errors are raised at parse time — consistent with the existing `--version` + `--spec`
check — so callers get a clear message regardless of how `main()` dispatches.

## Tests added

- `test_explore_and_review_mutually_exclusive` — `_parse(["--explore", "--review"])` exits 1
- `test_explore_and_version_mutually_exclusive` — `_parse(["--explore", "--version", "v5"])` exits 1

## Rules for future work

1. Any flag that implies a specific execution path (host-only, container, version loop)
   should have explicit mutual-exclusion checks against incompatible flags in `_parse()`,
   not rely on dispatch order in `main()` to suppress them silently.
2. Add a mutual-exclusion test whenever a new flag is added that conflicts with an
   existing one.
