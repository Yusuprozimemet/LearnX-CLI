# fix010 — Slide subtitle showed "Â·" (UTF-8 mojibake) on title card

**Date:** 2026-05-09

## Symptom

Title card subtitle rendered as `4 units Â· beginner` instead of
`4 units · beginner`.

## Root cause

`_build_title_card()` built the subtitle string with `·` (U+00B7 MIDDLE DOT).
`tutorial.visuals.json` was written with `ensure_ascii=False, encoding="utf-8"`
so the file stored the raw UTF-8 bytes `\xC2\xB7`. When the JSON was later
read back without an explicit encoding (defaulting to Windows CP1252), `\xC2`
decoded as `Â` and `\xB7` decoded as `·`, producing the mojibake string that
was then passed to Pillow's `draw.text()`.

## Files changed

- `tutor/generation/visual_planner.py` — `_build_title_card()`

## What changed

Replaced the Unicode middle dot with an ASCII hyphen:

```python
# before
subtitle=f"{n} unit{'s' if n != 1 else ''} · beginner"
# after
subtitle=f"{n} unit{'s' if n != 1 else ''} - beginner"
```

This is safe for all console encodings and avoids the need to enforce UTF-8
everywhere the JSON file is consumed. The existing `tutorial.visuals.json`
cache was also patched in place.
