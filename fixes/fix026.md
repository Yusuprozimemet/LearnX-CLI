# fix026 — Source text truncated: max_source_tokens was too low

**Date:** 2026-05-15

## Symptom

Generation logged this warning and silently cut the source material short:

```
Source text truncated from 2965 to 2307 words for context limit
```

The dialogue was generated from an incomplete version of the document — concepts
from later sections were missing entirely.

## Root cause

`tutor/llm_config.toml` had `max_source_tokens = 3000`. The truncation formula in
`dialogue.py` is:

```python
max_words = int(max_tokens / 1.3)   # 3000 / 1.3 = 2307 words
```

A source document with 2965 words requires ~3855 tokens — above the 3000 limit.

## Fix

Raised `max_source_tokens` from 3000 to 8000 in `tutor/llm_config.toml`:

```toml
[limits]
max_source_tokens = 8000   # was 3000
```

8000 tokens = ~6150 words max — well above any current document size.
Groq's llama-3.3-70b-versatile has a 128k context window, so this is safe.
OpenRouter models tested (laguna-xs.2, owl-alpha) also handle this comfortably.

## Rules for future work

1. If truncation warnings appear, raise `max_source_tokens` in `llm_config.toml`
   — no code change needed.
2. The formula is always `max_words = int(max_source_tokens / 1.3)`. Work backwards
   from your longest expected document word count to set the right limit.
3. Groq's free tier enforces a per-minute token rate limit, not a per-request context
   limit — raising this value does not cause 413 errors on groq.
