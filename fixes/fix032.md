# fix032 — run_explore() raises FileNotFoundError on fresh clones

**Date:** 2026-05-16

## Symptom

Running `python scripts/learnx_dk.py --explore` on a fresh clone (or any machine
where Claude Code has never been run in the project directory) raised:

```
FileNotFoundError: [Errno 2] No such file or directory: '.claude/settings.local.json'
```

The process exited immediately, before Claude was launched, and the temp permissions
file was never written.

## Root cause

`SETTINGS_LOCAL = pathlib.Path(".claude/settings.local.json")`

`run_explore()` called `SETTINGS_LOCAL.write_text(...)` directly. `pathlib.Path.write_text()`
does not create missing parent directories — it raises `FileNotFoundError` if the
parent (`.claude/`) does not exist.

On a fresh clone the `.claude/` directory is git-ignored and therefore absent.

## Fix

Added one line before the `write_text()` call in `run_explore()`:

```python
SETTINGS_LOCAL.parent.mkdir(parents=True, exist_ok=True)
SETTINGS_LOCAL.write_text(json.dumps(EXPLORE_PERMISSIONS, indent=2))
```

`exist_ok=True` makes the call a no-op when `.claude/` already exists, so there
is no regression for users who have previously run Claude in the project.

## Test added

`test_explore_writes_settings_local_when_claude_dir_absent` — monkeypatches
`SETTINGS_LOCAL` to a path whose parent directory does not exist, calls
`run_explore()`, and asserts no exception is raised and cleanup still runs.

## Rules for future work

1. Never call `Path.write_text()` (or `open(..., "w")`) on a path whose parent
   directory is not guaranteed to exist. Always call `parent.mkdir(parents=True,
   exist_ok=True)` first when writing to paths that may be git-ignored or
   user-specific.
2. Any function that writes a temporary file should have a test that exercises the
   case where the parent directory is absent.
