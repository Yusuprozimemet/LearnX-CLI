# fix040 — Parameters received but silently discarded: extra_args hardcoded, agents_dir placeholder missing

**Date:** 2026-05-16

## Symptom

Extra CLI arguments passed to `run_phase1()` / `run_phase2()` (e.g. `--model`, custom
`--max-turns`) had no effect on the Docker/Claude command that was built. Additionally,
the `agents_dir` value was always absent from the phase 2 prompt, so the review agent
had no path to look for agent files — even when `--agents-dir` was passed explicitly.

## Root cause

Two independent "received but discarded" bugs in `run_review.py`:

**Bug 1 — hardcoded `extra_args=[]` in both phase functions:**
```python
# before — in run_phase1() and run_phase2()
cmd = build_command(project_dir, home_dir, extra_args=[], interactive=False)
#                                          ^^^^^^^^^^^^ caller's value ignored
```
Both functions declared `extra_args: list[str]` in their signature but then hardcoded
`extra_args=[]` when calling `build_command()`. The caller's value was silently replaced.

**Bug 2 — `{agents_instruction}` placeholder missing from `PHASE_2_PROMPT_TEMPLATE`:**
```python
# before
agents_instruction = f"Review agents are in {agents_dir}/."
prompt = PHASE_2_PROMPT_TEMPLATE.format(
    phase1_report=phase1_report,
    agents_instruction=agents_instruction,   # KeyError — placeholder absent
)
```
`run_phase2()` computed `agents_instruction` correctly but `PHASE_2_PROMPT_TEMPLATE` had
no `{agents_instruction}` in its body. The value was computed and dropped.

## Fix

**Bug 1:**
```python
# after
cmd = build_command(project_dir, home_dir, extra_args=extra_args, interactive=False)
```

**Bug 2** — added the placeholder to the template body:
```
{agents_instruction}

Launch these two agents IN PARALLEL using the Task tool:
```

## Tests added

- `test_phase2_prompt_template_has_phase1_report_placeholder` extended to assert
  `"{agents_instruction}" in PHASE_2_PROMPT_TEMPLATE`
- Existing `test_build_review_command_accepts_custom_agents_dir` and
  `test_review_main_dry_run_accepts_agents_dir_flag` already cover the agents_dir path

## Rules for future work

1. Never replace a function parameter with a literal inside the function body. Forward
   the parameter. (This rule is now in CLAUDE.md under "Code Quality Rules".)
2. After adding a new `format()` call, verify every keyword argument has a matching
   `{placeholder}` in the template string. A missing placeholder is not a runtime error
   until the format call executes — it silently computes and discards the value.
3. When writing a function that wraps another, grep for every argument the inner
   function accepts and confirm each one is forwarded, not hardcoded.
