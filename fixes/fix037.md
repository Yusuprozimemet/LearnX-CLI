# fix037 — `--wait 0` (and `--session-timeout 0`, `--idle-timeout 0`) silently ignored

**Date:** 2026-05-16

## Symptom

Passing `--wait 0` to disable rate-limit retries had no effect. The value `0`
was treated as "not provided" and the config default (`rate_limit_wait_minutes = 2`)
was used instead. Same bug affected `--session-timeout 0` and `--idle-timeout 0`.

## Root cause

`main()` used Python truthiness to select between the CLI value and the config default:

```python
rate_limit_wait_s = (wait_min or res["rate_limit_wait_minutes"]) * 60.0
```

`_extract_int_flag` returns `(0, rest)` when `--wait 0` is given. `0 or default`
evaluates to `default`, so the explicit zero is silently discarded.

The same pattern appeared for `--session-timeout` and `--idle-timeout`, making all
three zero-values unreachable from the CLI.

## Fix

Replaced truthiness with an explicit `None` check for all three flags:

```python
# before
session_timeout_s = (session_timeout_min or res["session_timeout_minutes"]) * 60.0
idle_timeout_s    = (idle_timeout_min    or res["idle_timeout_minutes"])     * 60.0
rate_limit_wait_s = (wait_min           or res["rate_limit_wait_minutes"])   * 60.0

# after
session_timeout_s = (
    res["session_timeout_minutes"] if session_timeout_min is None else session_timeout_min
) * 60.0
idle_timeout_s = (
    res["idle_timeout_minutes"] if idle_timeout_min is None else idle_timeout_min
) * 60.0
rate_limit_wait_s = (
    res["rate_limit_wait_minutes"] if wait_min is None else wait_min
) * 60.0
```

## Test added

`test_wait_zero_overrides_config_default` — patches `run_yolo_version` and asserts
that `main(["--version", "v5", "--wait", "0"])` passes `rate_limit_wait_s=0.0`.

## Rules for future work

1. Never use `x or default` to select between an optional int and a fallback —
   use `default if x is None else x`. Zero is a valid and meaningful integer.
2. Any `_extract_int_flag` caller must use `is None` for the absence check.
3. Apply this rule to all three resilience flags (`--session-timeout`, `--idle-timeout`,
   `--wait`) whenever new numeric CLI flags are added in this pattern.
