# fix002 — 413 "Request too large" from Groq free tier

**Date:** 2026-05-08

## Symptom

```
Error: LLM call failed after retry: Error code: 413 - {'error': {'message':
'Request too large for model `llama-3.1-8b-instant` ... Limit 6000, Requested 6916 ...'}}
```

## Spec gap

The spec set `MAX_SOURCE_TOKENS = 4000` without accounting for Groq's combined
(input + output) per-request cap. No constraint stated the total token budget.
Future specs that add LLM call types must specify both the input token limit and the
expected output size, and verify the sum fits within the provider's request cap.

## Root cause

Groq's free tier caps each request (input + output tokens combined) at 6 000 tokens for
`llama-3.1-8b-instant`. The dialogue prompt was oversized:

| Part | Tokens |
|---|---|
| `dialogue.txt` system prompt + speaker constraint | ~630 |
| Unit JSON | ~200 |
| Source text (`MAX_SOURCE_TOKENS = 4 000`) | ~4 000 |
| **Input total** | **~4 830** |
| Expected output (uncapped) | ~2 000+ |
| **Request total** | **~6 830 → exceeded 6 000** |

## Files changed

- `tutor/llm_config.toml` — `max_source_tokens`, `max_tokens` section
- `tutor/infra/llm.py` — pass `max_tokens` to API, handle 413 as non-retriable
- `tutor/ingestion/summarizer.py` — truncate chunk text before summarising

## What changed

1. **`max_source_tokens` 4 000 → 1 500** — keeps dialogue input ~2 330 tokens.
2. **Explicit `max_tokens` per call type** passed to the API:
   `curriculum 2 000 / dialogue 1 500 / summarize 400 / qa 600`.
3. **413 now raises immediately** — retrying an oversized prompt wastes quota; error message points to `llm_config.toml`.
4. **Chunk text truncated** to `max_summarize_input_tokens = 3 000` before summarisation.

**Token budget per dialogue call after fix:**
```
Input:  ~630 + ~200 + ~1 500 = ~2 330
Output: 1 500 (capped)
Total:  ~3 830  ← safely under 6 000
```
