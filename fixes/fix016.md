# fix016 — CLI code quality: split commands.py, remove dead code, clean up imports

## Problem

`tutor/cli/commands.py` had grown to **621 lines**, far exceeding the project's 400-line limit.
It mixed four distinct concerns in a single file:
- `ShellContext` dataclass (shared state)
- Playback command handlers (10 functions + 3 helpers)
- Pipeline command handlers (generate, sessions, inspect, dry-run)
- Shell utility commands (help, clear) and the dispatch table

Additional issues found during audit:
- Dead code: unreachable `print()` at line 254 of `_format_duration()` (after `return`)
- Lazy `import logging as _logging` inside `_apply_log_level()` despite `logging` already being
  a top-level import — shadowed its own module alias unnecessarily
- `import datetime`, `import json`, `import os` scattered as inline lazy imports inside handlers
  with no circular-import reason to be lazy
- `_normalize_audio()` in `video_assembler.py` was completely dead (never called); it was the
  old two-pass loudnorm approach replaced in fix009 by a single `volume=5dB` encode flag
- `_write_session_meta` logic was inlined as a 20-line block inside `cmd_generate()`,
  obscuring the handler's intent

## Fix

### New files

**`tutor/cli/shell_context.py`** (17 lines)
- Extracted `ShellContext` dataclass out of `commands.py`
- Breaks the circular-import risk that would otherwise arise between `commands.py` and
  the new `playback_commands.py`

**`tutor/cli/playback_commands.py`** (239 lines)
- Extracted all playback-related code: `_require_player`, `_get_flag`, `_resolve_units_dir`,
  and the ten command handlers: `cmd_play`, `cmd_pause`, `cmd_resume`, `cmd_stop`, `cmd_next`,
  `cmd_prev`, `cmd_replay`, `cmd_ask`, `cmd_summary`, `cmd_status`
- Imports `ShellContext` from `shell_context.py` — no circular dependency

### Modified files

**`tutor/cli/commands.py`** (621 → 383 lines)
- Removed all playback commands (now imported from `playback_commands.py`)
- Re-exports `ShellContext` from `shell_context.py` so all existing callers
  (`shell.py`, `video_commands.py`, tests) continue to work unchanged
- Moved `import datetime`, `import json`, `import os` to top-level imports
- Fixed dead code: removed unreachable `print()` after `return` in `_format_duration()`
- Fixed redundant lazy `import logging as _logging` in `_apply_log_level()` — now uses the
  already-imported `logging` module directly
- Extracted `_write_session_meta(output, input_path)` helper to isolate meta-JSON writing
  from the `cmd_generate` handler

**`tutor/visual/video_assembler.py`** (336 → 289 lines)
- Removed dead function `_normalize_audio()` — the two-pass loudnorm approach it implemented
  was superseded by the `volume=5dB` audio filter in `_build_unit_video()` (see fix009)
  and was never called from anywhere in the codebase

### Unchanged callers

`shell.py`, `video_commands.py`, `tutor/tests/cli/test_commands.py`,
`tutor/tests/cli/test_video_commands.py` — all import `ShellContext`, `COMMAND_MAP`,
`_format_duration`, `_read_meta`, `cmd_sessions`, `cmd_ask` from `tutor.cli.commands`
and continue to work without modification.
